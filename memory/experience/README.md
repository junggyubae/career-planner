# experience/

Your history — research, work, projects, education, awards.

- `_manager.md` — **the spine**: a time-ordered index of every experience with a one-line summary and a link to its detail file. Always kept in sync.
- One markdown file per experience (e.g. `2024-summer-lab-x.md`).

**How it grows:** Info Retrieval's *upload path* creates new files from a CV; its *interview path* refines and enriches an existing file with deeper detail. Read by **Finder** (fit) and **Alignment** (CV/SOP evidence).

This folder is **private** (git-ignored). Only this README is committed.

## File convention (per experience)

```markdown
# <Title> — <Organization>
- **Type:** research | work | project | education | award
- **Dates:** YYYY-MM – YYYY-MM (or "present")
- **Location:**
- **Advisor / manager:**
- **One-line:**

## What I did
- ...

## Impact / outcomes
- ...

## Skills & tools
- ...

## Links
- paper / repo / demo / page
```
