# action/discovery/ — Finder + Paper Briefing

Use one generated index plus one folder per PI/lab:

```text
INDEX.md
<school-slug>-<pi-slug>/
  finder.md
  dossier.md
  briefing.md
  notes.md
```

- `INDEX.md`: generated private overview across schools, PIs, labs, fit bands,
  briefing status, alumni/outcome signals, connection signals, and next actions.
- `finder.md`: the verified PI/lab profile from Finder.
- `dossier.md`: candidate papers, links, citations, abstracts, and access notes.
- `briefing.md`: abstract-based paper notes, fit analysis, interview questions,
  and SOP talking points.

Do not save full copyrighted PDFs here by default. Store links, citations, and
notes instead. Full-paper reading should happen only when the user explicitly
asks for a paper summary or specific details inside a paper.
